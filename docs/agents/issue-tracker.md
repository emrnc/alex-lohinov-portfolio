# Issue Tracker: Linear

Issues and product work for this repo live in Linear.

## Conventions

- Use Linear as the source of truth for issues, planning, and triage.
- When a skill says "publish to the issue tracker", create or update the relevant Linear issue.
- When a skill says "fetch the relevant ticket", read the referenced Linear issue and use its title, description, comments, status, labels, and linked context.
- Use the triage vocabulary in `docs/agents/triage-labels.md` when applying labels or equivalent Linear states.

## Notes

If a Linear workspace, team, project, or status workflow is not specified in the prompt, ask for the missing detail before writing to Linear.
